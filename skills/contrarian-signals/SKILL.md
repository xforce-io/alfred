# 每日反共识信号 (Daily Contrarian Signals)

基于当日财经热点新闻，生成反共识的投资逻辑和深度分析，挑战市场主流叙事。

## When to Use

- 定时任务（每日20:00）自动推送
- 用户主动要求"反共识分析"、"逆向思考"、"市场共识是什么"
- 用户提到某个热点新闻，想听不同角度的分析

## 关键执行约束（必须遵守）

1. **完整性优先于数量和篇幅**：默认生成 **3 个反共识信号**。只有当用户明确要求且输出空间充足时，才生成4-5个。不得因追求“完整版”导致报告截断。
2. **禁止任务摘要/执行摘要前置**：最终输出必须直接从 `# 每日反共识信号` 开始，不要添加“任务执行结果摘要”“核心结论概览”等额外内容，避免挤占正文空间。
3. **禁止中途截断**：发送前必须确认每个信号都包含完整的「市场共识 / 反共识逻辑 / 资产影响 / 置信度 / 数据依据」，且文末包含免责声明。
4. **如果输出空间不足**：优先压缩单条篇幅，而不是输出未完成内容。压缩顺序：
   - 将4-5个信号减少为3个；
   - 每条反共识逻辑压缩为2段以内；
   - 每段控制在120-180字；
   - 资产影响每类最多列2个方向。
5. **技能加载容错**：如果外部调度指令要求先调用 `_load_resource_skill("contrarian-signals")`，但该调用失败或资源列表未注册，不得直接判定任务失败；应视为当前技能说明已可用，继续执行新闻获取和分析流程。
6. **数据不可编造**：任何具体数字、历史回测、百分比、时间窗口、公司财务指标，必须来自新闻原文、抓取结果或明确可公开核验的来源。若无法核验，不得给出具体数值，只能使用定性表述，并标明“未见可核验数据支撑”。

## 执行流程

### Step 1: 获取今日新闻

优先执行：

```bash
cd ~/dev/github/alfred && .venv/bin/python skills/gray-rhino/scripts/news_fetcher.py --max-age 24 --format json
```

若脚本执行失败：
- 不要编造“已获取新闻条数”；
- 可基于用户提供的新闻或当前会话中已有新闻继续分析；
- 若没有任何可用新闻，应明确说明“缺少可核验的当日新闻输入，无法生成合规反共识信号”，而不是虚构热点。

### Step 2: 生成反共识分析

基于获取的新闻数据，按以下框架生成 **默认3个反共识信号**：

#### 分析框架

对每条重要新闻，识别：
1. **市场共识**：大多数人看到这条新闻后的直觉反应
2. **反共识逻辑**：为什么直觉可能是错的，用新闻事实、公开数据、历史经验或结构性因素反驳
3. **资产影响**：如果反共识逻辑成立，哪些资产会受益/受损
4. **数据依据**：列出使用到的新闻来源或可核验事实，避免无来源数字

#### 反共识思维工具箱

- **时间维度错配**：市场关注短期冲击，忽略长期结构性变化
- **二阶效应**：A导致B，但B又导致C，C才是真正的影响
- **均值回归**：极端情绪/估值终将回归
- **幸存者偏差**：只看到赢家，忽略了更多失败案例
- **反身性**：预期本身改变了现实（索罗斯框架）
- **边际定价者切换**：谁在买/卖决定了价格，而非基本面
- **历史类比陷阱**："这次和XX年一样"往往忽略关键差异

### Step 3: 输出长度控制

为避免报告被截断，必须遵守以下篇幅上限：

- 全文建议控制在 **1800-2600个中文字**；
- 默认 **3条信号**；
- 每条信号：
  - 标题：不超过28个中文字；
  - 市场共识：1句话；
  - 反共识逻辑：2段以内，每段不超过180个中文字；
  - 资产影响：利好/利空各不超过2项；
  - 数据依据：1行，列新闻来源/事实来源，不展开长篇引用。

如用户要求“完整版分析”，也必须保证完整收尾；“完整版”意味着逻辑链条完整，而不是无限扩写。

## 质量要求

1. **禁止编造数据**：所有数据引用必须来自新闻原文或公开可查来源；无法核验的数据不得出现。
2. **禁止伪历史回测**：不得凭空写“过去N次平均涨跌X%”。只有在新闻抓取结果或可靠公开来源中明确存在该统计时，才可使用。
3. **逻辑严密**：反共识不是抬杠，需要有清晰因果链条：共识为何形成 → 哪个前提可能错 → 若错会如何影响资产定价。
4. **覆盖全资产**：A股/港股、美股、商品/外汇/利率均可涉及；默认3条信号应尽量覆盖不同资产类别，避免全部集中在单一主题。
5. **深度优先但不得冗长**：每条信号必须有完整逻辑推演，但不要堆砌长篇背景，避免截断。
6. **避免陈词滥调**："长期看好"、"逢低买入"、"风险偏好修复"等空话不算反共识。
7. **避免过度具体交易建议**：可分析“利好/利空资产方向”，但不要给出确定性买卖点、目标价或收益承诺。
8. **不得虚构执行结果**：如果没有真实执行新闻抓取，不得声称“成功获取213条新闻”等。
